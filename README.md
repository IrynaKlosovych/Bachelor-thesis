# Симулятор аналізу та прогнозу результатів виборів

**Кваліфікаційна робота бакалавра** 

**Спеціальність:** 121 «Інженерія програмного забезпечення»

## Живий застосунок
**URL:** https://bachelor-thesis-frontend-one.vercel.app/

## Мета і завдання

**Мета випускної кваліфікаційної роботи бакалавра:** розробити симулятор аналізу та прогнозування результатів виборів на основі *характеристик представників електорату*, *зовнішніх чинників* (що можуть описувати стан країни), а також таких елементів політичної системи як *виборча система*.

**Завданнями для реалізації мети є:**
1. Розгляд та аналіз аналогів системи, що розробляється, з метою дослідження функціоналу, переваг та недоліків.
2. Визначення архітектури системи.
3. Проєктування системи та вибір засобів реалізації.
4. Розробка алгоритмів системи.
5. Реалізація функціоналу.
6. Тестування системи на відповідність поставлених вимог і задач.

## Об'єкт і предмет дослідження
**Об'єктом дослідження** є процес створення симуляції аналізу і прогнозу результатів виборів.

**Предметом дослідження** є методи і засоби для реалізації мети роботи; розробка алгоритмів та функціоналу системи визначення ймовірних результатів виборів з урахуванням характеристик електорату, зовнішніх факторів, виборчої системи.

**Практичне значення** полягає в розробці системи, що зможе бути використана в освітніх цілях під час вивчення політичних і соціальних дисциплін та стане прототипом складнішої системи для наукових досліджень в
сфері соціології. В галузі розробки ПЗ результати можуть бути використанні для перевірки доцільності і якості розроблених рішень для прогнозування соціальних процесів з подальшим їх вдосконаленням.

## Архітектура проєкту
Гібридна архітектура, що включає клієнстьку частину на основі компонентів, що через REST API зв'язується з монолітним сервером. Сервер містить шар API для прийняття запитів, шар бізнес-логіки та ядро з реалізованими модулями дослідження вподобань виборців і виборчих систем.

**В проєкті реалізовано:**

## Інструкція запуску

**Здійсність клонування репозиторію**

Через SSH:
```
GitLab:
git clone git@git.ztu.edu.ua:ipz/2022-2026/ipz-22-1/klosovych-iryna/bachelor-thesis.git

GitHub:
git clone git@github.com:IrynaKlosovych/Bachelor-thesis.git
```
Через HTTPS
```
GitLab:
git clone https://git.ztu.edu.ua/ipz/2022-2026/ipz-22-1/klosovych-iryna/bachelor-thesis.git

GitHub:
git clone https://github.com/IrynaKlosovych/Bachelor-thesis.git
```

**Перейдіть у папку для серверу**
```
cd backend
```
**Активуйте віртуальне середовище та встановіть залежності**
```
pip install -r requirements.txt
```
**Запустити локально**
```
python manage.py runserver 
```

**Перейдіть у папку для клієнту**
```
cd frontend
```
**Встановіть залежності**
```
npm install
```
**Запустити локально**
```
npm run dev
```
